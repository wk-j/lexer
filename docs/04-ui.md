# Webview / UI

## Technology Choice

Use **Tauri v2** to embed a system webview in a native window. Tauri avoids bundling a full browser engine (like Electron) while providing a mature framework for desktop apps with built-in IPC, window management, native menus, file dialogs, auto-update, and CLI scaffolding.

**Why Tauri:**

- Built-in command system for type-safe Rust <-> JS IPC
- Native menu bar, file dialogs, and system tray support out of the box
- Tauri v2 plugin ecosystem (fs, dialog, shell, clipboard, etc.)
- Built-in auto-updater for distribution
- Uses `wry` under the hood -- same lightweight webview, higher-level API
- Strong cross-platform support (macOS, Linux, Windows)
- Active ecosystem and documentation

## Frontend

The frontend is a single HTML page injected into the webview. It consists of:

- **HTML template**: Skeleton with a `<div id="content">` container
- **CSS stylesheet**: Typography, code block styling, responsive layout, themes
- **JS glue**: Minimal JavaScript to receive rendered HTML from Rust via IPC and inject it into the DOM

```
frontend/
  index.html       # Main HTML shell
  style.css        # Styling and themes
  app.js           # IPC listener, DOM manipulation
  themes/
    light.css      # Light theme
    dark.css       # Dark theme
```

## IPC Bridge (Tauri Commands & Events)

Communication between Rust and the webview uses Tauri's built-in command and event system.

**Webview -> Rust (Tauri Commands):**

Commands are type-safe Rust functions exposed to the frontend via the `#[tauri::command]` macro.

```rust
#[tauri::command]
fn open_file(path: String) -> Result<RenderedContent, String> {
    let content = fs::read_to_string(&path).map_err(|e| e.to_string())?;
    let html = render_markdown(&content);
    Ok(RenderedContent { html, title: filename(&path) })
}

#[tauri::command]
fn get_theme() -> String {
    // return current theme
}
```

```javascript
// JS side: invoke Rust command
const result = await window.__TAURI__.core.invoke('open_file', {
    path: '/path/to/file.md'
});
document.getElementById('content').innerHTML = result.html;
```

**Rust -> Webview (Tauri Events):**

Events push data from Rust to the frontend (e.g., file-change notifications).

```rust
// Rust side: emit event to frontend
app_handle.emit("file-changed", RenderedContent { html, title })?;
```

```javascript
// JS side: listen for events
const { listen } = window.__TAURI__.event;
listen('file-changed', (event) => {
    document.getElementById('content').innerHTML = event.payload.html;
});
```

**Defined Commands:**

| Command            | Args               | Returns              | Description                      |
| ------------------ | ------------------ | -------------------- | -------------------------------- |
| `open_file`        | `path: String`     | `RenderedContent`    | Parse and render a Markdown file |
| `open_file_dialog` | none               | `Option<RenderedContent>` | Show native file picker     |
| `list_themes`      | none               | `Vec<ThemeInfo>`     | List all built-in + user themes  |
| `load_theme`       | `name: String`     | `String` (CSS)       | Compile and return theme CSS     |
| `get_theme`        | none               | `String`             | Get current active theme name    |
| `set_theme`        | `name: String`     | `String` (CSS)       | Switch theme, return new CSS     |
| `get_theme_config` | `name: String`     | `Theme`              | Get full parsed theme for preview|
| `get_toc`          | none               | `Vec<TocEntry>`      | Get table of contents            |
| `set_effect`       | `effect: String, enabled: bool` | `()` | Toggle a visual effect      |
| `palette_search`   | `query: String, mode: String` | `Vec<PaletteItem>` | Fuzzy search for palette results |
| `palette_execute`  | `action: PaletteAction` | `()`            | Execute a palette action         |
| `scan_directory`   | `path: String`     | `Vec<FileEntry>`     | Recursively list .md files in dir|
| `get_recent_files` | none               | `Vec<FileEntry>`     | Get recently opened files        |
| `new_window`       | `path: Option<String>` | `String` (win id)| Open a new window (optionally with file) |
| `close_window`     | `window_id: String`| `()`                 | Close a specific window          |
| `list_windows`     | none               | `Vec<WindowInfo>`    | List all open windows            |
| `focus_window`     | `window_id: String`| `()`                 | Bring window to front            |
| `set_layout`       | `layout: String`   | `()`                 | Change layout of current window  |
| `get_layout`       | none               | `String`             | Get current window layout name   |

**Defined Events:**

| Event              | Payload            | Direction        | Description                       |
| ------------------ | ------------------ | ---------------- | --------------------------------- |
| `file-changed`     | `RenderedContent`  | Rust -> Frontend | File modified on disk, re-render  |
| `theme-updated`    | `String` (CSS)     | Rust -> Frontend | Active theme file changed on disk |
| `navigate`         | `{ path: String }` | Frontend -> Rust | User clicked a relative .md link |

## Styling Requirements

- Clean, readable typography (system font stack or configurable)
- Proper spacing for headings, paragraphs, lists, blockquotes
- Styled tables with borders and alternating row colors
- Code blocks with background, border-radius, and horizontal scroll
- Responsive: content reflows at different window sizes
- Full custom theme support (built-in + user-defined)
- Configurable font size
